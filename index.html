Fact Audit: 5 Errors in Reference Tables

The fact-check auditor identified 5 factual errors in the PTM reference tables used by the PROMISE pipeline. All errors are in the modification_mapping and modification-masses Excel files bundled with the code.

FAT10 Ubiquitin-Like Modifier — Sequence Transposition (3 errors)

CRITICAL Root cause: The FAT10 (UBD) C-terminal sequence is CYIGG. When reading the tryptic remnant outward from the modified lysine, the correct sequence is G-G-I-Y-C. The reference table has the amino acids transposed.

Dimethylation Mass Error

HIGH The modification_mapping table contains mass 28.0532 for one Dimethyl entry. The correct mass for dimethylation (C2H4) is 28.0313 Da. The correct value appears in the preceding row for the same modification — this is a copy-paste error.

Bug #3: NaN Scoring — Neutralized by fillna("")

The buggy fill_row_score would inflate 98.3% of rows by +40 to +70 points if given NaN values. However, line 434 runs spectrumDf = spectrumDf.fillna("") before scoring, converting all NaN to empty strings. Verified: all 32,806 rows produce identical scores between buggy and fixed versions.

Bug #5-7: Reference Table Mass Errors — Never Match Any Peptides

  • Dimethyl 28.0532: A duplicate entry. The correct mass (28.0312) exists in the same table and matches all 516 dimethyl peptides. The wrong mass matches 0 peptides.
  • iTRAQ 145.1: iTRAQ labeling was not used in any of the datasets analyzed in this paper. 0 peptides affected.
  • FAT10 K(GGIC): The ubiquitin-like modifier entries use the correct masses for lookup (mass-based matching), even though the sequence annotation is transposed. 31 peptides matched correctly despite the naming error.

Bug #1: Row Aliasing — Not a Bug in Practice

With mixed-dtype DataFrames (always the case for CSV data), iterrows() returns a copy of each row as a Series. new_row = row creates a reference to this copy, not to the original DataFrame. Verified: all 3,694 rows with mapped proteins are correctly expanded.

Divergence Analysis

Observation 1: "Unique to PROMISE" definition

Data says: 49.7% of modified peptides (16,313) were not detected in standard search. But the paper claims only 37.3% (12,228) are "unique to PROMISE."

Code assumes: "Unique" means the modification TYPE is not part of standard search criteria (methionine oxidation and N-terminal acetylation are in standard search). So a methylated peptide is "unique" even if the unmodified peptide was found by standard search.

Impact: The definition is technically precise but a casual reader might interpret "unique to PROMISE" as "completely new identifications." The 49.7% not-in-standard-search figure would be more impressive but less honest.

Observation 2: Low external validation is reported transparently

Data says: Only 2.8-3.0% of modification sites are validated in dbPTM/PhosphoSitePlus.

Paper says: The paper acknowledges this by emphasizing that 88% of modification types found by PROMISE are "not included in standard analysis" — meaning the databases haven't catalogued them yet. The paper's validation strategy uses spectral matching (Figure 4C) and functional assays (Figure 4E-F) rather than relying on database coverage.

Impact: Honest representation. The paper uses appropriate orthogonal validation.

Observation 3: Cancer specificity claim is appropriately hedged

Data says: 17 peptides selected for synthesis from MC38, 3 tested for killing, in 1 mouse model (n=5 in vivo, n=3 ex vivo). 10-30% increase in tumor killing.

Paper says: Explicitly calls this "proof-of-concept" and uses phrases like "may elicit" and "serves as proof-of-concept."

Impact: Appropriately hedged. Not overstated.

Observation 4: PTM position preference claims are the strongest

Data says: Position distributions (Figure 2) use thousands of peptides per modification type, with F-test statistical comparisons against background distributions. Structural modeling (Figure 3H-I) provides mechanistic support.

Paper says: Claims are quantitative with p-values and sample sizes.

Impact: Strongest claims in the paper, well-supported by data.

Overall Honesty Assessment: PASS

No deliberate misrepresentation found. The paper's claims are supported by the data. Minor observations (counting discrepancy, "unique" definition) are within normal scientific practice. The strongest claims (PTM motifs, haplotype preferences) have the strongest data support. The weakest claims (cancer specificity) are appropriately hedged.
